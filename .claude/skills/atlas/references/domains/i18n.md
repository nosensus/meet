# i18n

> **Location**: `src/i18n.ts` (+ `src/data/*.uz.ts`)
> **File Count**: 1 module + 3 parallel datasets
> **Confidence**: high

## Purpose

Bilingual support (Russian + Uzbek-latin) with no i18n library. Two concerns:

1. **UI chrome strings** — header, tabs, footers, button labels, `SayCard`/`CallLog`/`Blocks`
   labels. Held in `UI: Record<Lang, UIStrings>` in `src/i18n.ts`.
2. **Content** — the script graph, tips, and Ambax cards. Translated as **parallel datasets**
   (`src/data/*.uz.ts`) with identical keys/ids, selected by language in `App.tsx`.

## Key Files

| File | Purpose |
|------|---------|
| `src/i18n.ts` | `Lang = "ru" \| "uz"`, `LANGS` (switcher list), `DEFAULT_LANG`, `loadLang()`/`saveLang()` (localStorage), `UIStrings` interface, `UI` dictionary |
| `src/data/script.uz.ts` | `SCRIPT_UZ` — same node keys/option ids/`next` as `script.ts` |
| `src/data/tips.uz.ts` | `TIPS_UZ` + `TIP_CATEGORIES_UZ` |
| `src/data/ambax.uz.ts` | `AMBAX_UZ` + `AMBAX_CATEGORIES_UZ` |
| `src/components/PayScheme.tsx` | Carries its own `PAY: Record<Lang, PayContent>`; takes a `lang` prop |

## How it wires together

- `App.tsx` holds `lang` state (init from `loadLang()`), writes through `saveLang()` on change.
- `const t = UI[lang]` is passed to `StepCard` (→ `SayCard`), `CallLog`, and as `allLabel` to `Blocks`.
- Content datasets are chosen by `lang === "uz" ? *_UZ : *`.
- `Blocks` is given `key={`tips-${lang}`}` / `key={`ambax-${lang}`}` so its filter state resets
  when the language (and thus the category set) changes.

## Invariants

- **Key parity:** `SCRIPT_UZ` must have exactly the same node keys, option `id`s, and `next`
  targets as `SCRIPT`. Card `id`s must match across `tips.ts`/`tips.uz.ts` and `ambax.ts`/`ambax.uz.ts`.
- **Category consistency:** within one language, every card's `category` must exist in that
  language's `*_CATEGORIES` tuple.
- **Every `UIStrings` field** must be present for both `ru` and `uz` (TypeScript enforces this).
